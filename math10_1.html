Advanced Relations and Functions - Comprehensive Guide

1. Foundations of Relations

1.1 Formal Definition

A relation R from set A to set B is a subset of the Cartesian product A × B. We write aRb to denote that (a, b) ∈ R.

R ⊆ A × B = {(a, b) | a ∈ A, b ∈ B}

1.2 Classification of Relations

1.2.1 Binary Relations

A binary relation on a single set A is a relation from A to A (R ⊆ A × A).

1.2.2 Properties of Binary Relations

Property Definition Logical Form
Reflexive Every element relates to itself ∀a ∈ A, (a, a) ∈ R
Irreflexive No element relates to itself ∀a ∈ A, (a, a) ∉ R
Symmetric If a relates to b, then b relates to a ∀a, b ∈ A, (a, b) ∈ R ⇒ (b, a) ∈ R
Antisymmetric If a relates to b and b relates to a, then a = b ∀a, b ∈ A, (a, b) ∈ R ∧ (b, a) ∈ R ⇒ a = b
Asymmetric If a relates to b, then b does not relate to a ∀a, b ∈ A, (a, b) ∈ R ⇒ (b, a) ∉ R
Transitive If a relates to b and b relates to c, then a relates to c ∀a, b, c ∈ A, (a, b) ∈ R ∧ (b, c) ∈ R ⇒ (a, c) ∈ R
Total Any two elements are comparable ∀a, b ∈ A, (a, b) ∈ R ∨ (b, a) ∈ R

1.3 Special Types of Relations

1.3.1 Equivalence Relations

A relation R on A is an equivalence relation if it is reflexive, symmetric, and transitive.

Example: The relation "is congruent modulo n" on integers:

a ≡ b (mod n) if n divides (a - b)

This relation partitions ℤ into n equivalence classes: [0], [1], ..., [n-1]

Proof that congruence modulo n is an equivalence relation:

1. Reflexive: a ≡ a (mod n) since n divides (a - a) = 0

2. Symmetric: If a ≡ b (mod n), then n divides (a - b), so n divides (b - a), hence b ≡ a (mod n)

3. Transitive: If a ≡ b (mod n) and b ≡ c (mod n), then n divides both (a - b) and (b - c), so n divides their sum (a - c), thus a ≡ c (mod n)

1.3.2 Partial Orders

A relation R on A is a partial order if it is reflexive, antisymmetric, and transitive.

Example 1: The subset relation ⊆ on the power set P(S) of any set S.

Example 2: The divisibility relation | on ℕ: a|b if a divides b.

Hasse Diagram: A graphical representation of a partial order where:

  • Elements are represented as vertices
  • If x ≤ y, then x is drawn below y
  • Edges represent covering relations (x < y with no z such that x < z < y)

1.3.3 Total Orders

A partial order where every two elements are comparable (total relation).

Example: The standard ≤ relation on ℝ.

1.4 Operations on Relations

Inverse: R⁻¹ = {(b, a) | (a, b) ∈ R}

Composition: R ∘ S = {(a, c) | ∃b, (a, b) ∈ S ∧ (b, c) ∈ R}

Closures:

  • Reflexive closure: R ∪ {(a, a) | a ∈ A}
  • Symmetric closure: R ∪ R⁻¹
  • Transitive closure: R⁺ = ⋃ Rⁿ (n ≥ 1)

1.5 Applications of Relations

  • Database Theory: Relational algebra forms the foundation of SQL
  • Graph Theory: Relations represent edges in directed graphs
  • Computer Science: Used in automata theory, formal languages
  • Social Networks: Representing connections between individuals

2. Advanced Function Theory

2.1 Formal Definition

A function f: A → B is a relation from A to B satisfying:

  1. Totality: ∀a ∈ A, ∃b ∈ B such that (a, b) ∈ f
  2. Uniqueness: If (a, b) ∈ f and (a, c) ∈ f, then b = c

We write f(a) = b for (a, b) ∈ f.

2.2 Classification of Functions

Type Definition Logical Form Example
Injective (1-1) Distinct inputs map to distinct outputs ∀a₁, a₂ ∈ A, f(a₁) = f(a₂) ⇒ a₁ = a₂ f(x) = 2x on ℝ
Surjective (Onto) Every element of B is mapped to ∀b ∈ B, ∃a ∈ A such that f(a) = b f(x) = x³ on ℝ → ℝ
Bijective Both injective and surjective Both conditions above f(x) = x on ℝ
Partial Not defined for all elements of A Domain ⊂ A f(x) = 1/x on ℝ
Constant Output is the same for all inputs ∃c ∈ B, ∀a ∈ A, f(a) = c f(x) = 5
Identity Maps each element to itself ∀a ∈ A, f(a) = a id_A: A → A

2.3 Function Operations

Composition: (g ∘ f)(x) = g(f(x))

Properties:

  • Associative: h ∘ (g ∘ f) = (h ∘ g) ∘ f
  • Identity: f ∘ id = id ∘ f = f

Inverse: For bijective f, f⁻¹ is the unique function satisfying f⁻¹ ∘ f = id_A and f ∘ f⁻¹ = id_B

2.3.1 Invertibility Theorem

A function f: A → B has an inverse f⁻¹: B → A if and only if f is bijective.

Proof:

(⇒) If f has an inverse, then:

  • Injective: If f(a₁) = f(a₂), then a₁ = f⁻¹(f(a₁)) = f⁻¹(f(a₂)) = a₂
  • Surjective: For any b ∈ B, f(f⁻¹(b)) = b

(⇐) If f is bijective, define f⁻¹(b) = the unique a such that f(a) = b

2.4 Special Functions

2.4.1 Polynomial Functions

f(x) = aₙxⁿ + aₙ₋₁xⁿ⁻¹ + ... + a₁x + a₀

Properties:

  • Degree: Highest power with non-zero coefficient
  • Roots: Solutions to f(x) = 0
  • Fundamental Theorem of Algebra: Every non-constant polynomial has at least one complex root

2.4.2 Exponential and Logarithmic

f(x) = aˣ (a > 0, a ≠ 1) and its inverse f⁻¹(x) = logₐx

Properties:

  • aˣ⁺ʸ = aˣaʸ
  • logₐ(xy) = logₐx + logₐy
  • Change of base: logₐb = log_c b / log_c a

2.4.3 Trigonometric Functions

sin(x), cos(x), tan(x), etc., with various identities and periodic properties

2.5 Function Spaces

The set of all functions from A to B is denoted Bᴬ or A → B.

For finite sets: |Bᴬ| = |B|^|A|

Important function spaces:

  • C(ℝ): Continuous real-valued functions
  • L²(ℝ): Square-integrable functions
  • ℤ → ℤ: All integer sequences

2.6 Applications of Functions

  • Cryptography: One-way functions in public-key systems
  • Physics: Describing physical relationships (e.g., position vs. time)
  • Computer Science: Lambda calculus, functional programming
  • Economics: Utility functions, production functions

3. Advanced Topics and Theorems

3.1 Cantor's Theorem

For any set A, there is no surjection from A to its power set P(A).

Proof by diagonalization:

Assume f: A → P(A) is surjective. Let D = {a ∈ A | a ∉ f(a)}.

Since f is surjective, ∃d ∈ A such that f(d) = D.

Question: Is d ∈ D?

  • If d ∈ D, then by definition of D, d ∉ f(d) = D (contradiction)
  • If d ∉ D, then d ∉ f(d), so by definition d ∈ D (contradiction)

Thus no such f can exist.

Corollary: There are different sizes of infinity. The power set of ℕ is uncountable.

3.2 Schröder-Bernstein Theorem

If there exist injections f: A → B and g: B → A, then there exists a bijection h: A → B.

Application: Show |[0,1]| = |(0,1)| by constructing injections:

f: (0,1) → [0,1] is the identity

g: [0,1] → (0,1) via g(x) = x/2 + 1/4

3.3 Axiom of Choice and Zorn's Lemma

The Axiom of Choice: For any collection of non-empty sets, there exists a function (choice function) that selects an element from each set.

Zorn's Lemma: If every chain in a partially ordered set has an upper bound, then the set contains at least one maximal element.

These are equivalent in ZF set theory and have profound implications for infinite sets.

3.4 Fixed Point Theorems

Banach Fixed Point Theorem: If f: X → X is a contraction on a complete metric space, then f has exactly one fixed point.

Brouwer Fixed Point Theorem: Any continuous function from a convex compact subset of ℝⁿ to itself has at least one fixed point.

Applications: Nash equilibria in game theory, differential equations, computer science (program semantics)

3.5 Cardinality of Function Spaces

For infinite sets A and B:

  • If |A| = κ and |B| = λ ≥ 2, then |Bᴬ| = λ^κ
  • |ℝ^ℕ| = |ℝ| (sequences of real numbers)
  • |ℕ^ℕ| = |ℝ| (all infinite sequences of natural numbers)

3.6 Function Composition Algebra

The set of all functions f: A → A forms a monoid under composition:

  • Associative operation (composition)
  • Identity element (identity function)

If we restrict to bijections, we get the symmetric group S_A.

4. Computational Aspects

4.1 Representing Relations in Code

// Relation as a set of pairs in Python
relation = {(1, 'a'), (2, 'b'), (3, 'c')}

// Checking properties
def is_reflexive(relation, set_A):
    return all((x, x) in relation for x in set_A)

def is_symmetric(relation):
    return all((y, x) in relation for (x, y) in relation)

// Transitive closure (Warshall's algorithm)
def transitive_closure(relation):
    closure = set(relation)
    elements = {x for (x, y) in relation} | {y for (x, y) in relation}
    for k in elements:
        for i in elements:
            for j in elements:
                if (i, k) in closure and (k, j) in closure:
                    closure.add((i, j))
    return closure

4.2 Function Composition in Programming

// JavaScript function composition
const compose = (...fns) => x => fns.reduceRight((acc, fn) => fn(acc), x);

const double = x => x * 2;
const add5 = x => x + 5;

const doubleThenAdd5 = compose(add5, double);
console.log(doubleThenAdd5(3)); // 11
